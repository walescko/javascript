Grafo: é um modelo abstrato de uma estrutura de rede que contém nós/vértices (nodes/vertice) conectados por arestas (edges).

Um grafo é composte de um conjunto de vértices e arestas.

Os vértices conectados por uma arestas são vértices adjacentes. E no grau de um vértice corresponde ao número de vértices adjacente.

Um caminho (path) é a sequência de vértices consecutivos. Um caminho simples não contém vértices repetidos. Um ciclo é um caminho simples, exceto pelo último vértice que é igual ao primeiro.

Um grafo acíclico é aquele que não tem ciclos. E um grafo conectado é aquele que há um caminho entre todos os pares de vértices.

Grafo direcionado é aquele que as arestas tem uma direção. Grafo não direcionado não tem direcionamento das arestas. Um grafo fortemente conectado o caminho é em ambas direções. E ainda os grafos podem ser sem peso ou com peso.



Represetando um grafo

A matriz de adjascências


A matriz de adjascência é a representação mais comum para os grafos. Cada nó é associado a um inteiro, que o índice do array (matriz). Se houber uma aresta entre um nó de índice i para um nó de índice j representamos como array[i][j] === 1, se não houver a representação fica como array[i][j] === 0.

Uma lista de adjacências é uma forma de representar grafos, sendo composta de uma lista de vértices adjacentes para cada vértice do grafo. Podemos usar as seguintes estruturas para lista de vértices adjacentes: array, lista ligada, mapa hash ou dicionário.

Matriz de Incidências

Um grafo também pode ser represetando por uma matriz de incidências. Nesse tipo de represetação cada linha representa um vertice (v) e cada coluna representa uma aresta (e - edge). A representação entre os dois objetos usa-se um array bidimencional, sendo o vértice v for incidente sobre a aresta e temos array[v][e]===1, caso não haja a incidência será array[v][e]===0. Esse tipo de matriz é usado para economicar espaço e mémoria.

Percorrendo Grafos

Breadth-First Search - Busca em Largura:Armazena os vértices em uma fila, os vértices não explorados mais antigos serão explorados antes.

Depth-First Search - Busca em Profundidade:Armazena os vértices em uma pilha, os vértices serão explorados ao longo de um caminho, visitando um novo vértice adjacente se houver um disponível.
